SpringBoot

发表于 2017-06-24   |   分类于 日志 ， SpringBoot

前言
最近很多地方都在讨论SpringBoot这个开源框架，并且发现很多公司开发都在使用，现在的它的热度也是越来越高，所以决定用点时间来了解和学习，为自己充充电。


正文
个人感觉SpringBoot是属于一种轻量级的、适合快速开发的一个框架。
从本质上来说他其实还是属于Spring，但是它现在能代替你去书写原来Spring的一些配置，让整个项目能够快速的实现完成并且运行起来，让你几乎很少甚至不用去管理Sprign的配置。


SpringBoot核心

* 自动配置（auto-configuration）：可以针对许多Spring常用应用功能，SpringBoot能够自动提供相关配置。
- 启动依赖（starters）：如果需要其他功能，告诉SpringBoot，它就能引入需要的库。
- 命令行界面（CLI）：这是一个非必选的特性，只需要写代码就能完成完整的应用，无需构建。
- 系统集成（actuator）：提供系统的自省和监控的集成功，可以查看配置和相关功能统计。

Pom.xml分析

SpringBoot和SpringMvc一样都有自己的maven的配置文件Pom.xml。
大致分为：
1.SpringBoot的父级依赖




这块可以说如果pom文件有这段配置，说明这个项目SpringBoot项目，这个parent是一个特殊的starter，提供所有的相关的Maven依赖，常用依赖包就可以省去version标签。
2.启动依赖




这一块SpringBoot提供了很多默认的依赖模块，说白了就是启动直接可以拿来用的功能，都是以Spring-boot-starter-XX 这种格式作为命名。
3.SpringbootMaven插件




这里SpringBootMaven插件提供了很多方便的功能。

- 打包项目成一个可执行的超级jar（Uber—JAR)，然后把应用程序的所有依赖打入JAR文件内，并且提供一个描述文件，其中的内容就可以让你使用java -jar来启动应用程序。
- 可以通过搜索public static void main()方法来标记可运行类。

SpringBoot入口

这里Demoapplication是非常重要的一个启动类，Springboot程序的入口就是在这里，这里直接用这个main方法来编写一下：
添加@RestController以及index方法


这里说明一下



- @SpringBootApplication是Springboot的核心注解，主要是开启自动配置的。


- @main方法这个就不多说了，java的应用main方法作为项目的入口。


- @RestControlller这个相当于@ResponseBody+@Controller的拼接结合，
注意！！！用这个注解的方法都以就送的格式进行输出。



项目启动有三种方式：


1. 常用的直接Run-DemoApplication的main方法。


2. 使用命令mvn Spring-boot：run在命令行启动。


3. 运行mvnpackage进行打包的时候，会打包成一个可以运行的jar文件，使用java -jar 可以直接运行。




———-

结尾

总结来说Springboot就是这么简单，一个demo或者说一个初始化的Springboot，添加一个控制类，就可以启动起来看看效果了，不需要任何配置文件。
作为了解Springboot必须要知道几点吧：
1. Springboot的父级依赖。
2. Spring-boot-starter-XX提供的许多默认依赖。
3. main入口的运用。
